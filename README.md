# Descripción breve de los programas desarrollados
# Programa 1: Programación Tradicional

En este programa se desarrolló un sistema sencillo para registrar y mostrar la información de una mascota utilizando variables y funciones. Los datos de la mascota (nombre, especie y edad) se ingresan mediante teclado y posteriormente se presentan de forma organizada. La solución se implementó sin utilizar clases ni objetos, siguiendo el paradigma de programación tradicional o estructurada.

# Programa 2: Programación Orientada a Objetos

En este programa se implementó una clase llamada Mascota, que contiene los atributos nombre, especie y edad, además de los métodos mostrar_informacion() y hacer_sonido(). A partir de esta clase se crearon dos objetos diferentes, permitiendo representar mascotas individuales con sus propias características y comportamientos. Esta solución aplica los principios básicos de la Programación Orientada a Objetos.

# Reflexión sobre las diferencias entre la Programación Tradicional y la Programación Orientada a Objetos

Durante el desarrollo de ambos programas se evidenció que la Programación Tradicional se centra principalmente en las funciones y en el flujo de instrucciones para resolver un problema. Este enfoque resulta adecuado para aplicaciones pequeñas y sencillas, ya que su implementación es directa y fácil de comprender.

Por otro lado, la Programación Orientada a Objetos organiza la información mediante clases y objetos, integrando tanto los datos como los comportamientos relacionados. Aunque inicialmente requiere una mayor planificación y comprensión de conceptos como clases, atributos y métodos, ofrece ventajas importantes en términos de organización, reutilización y mantenimiento del código.

En comparación, la programación tradicional es más simple para problemas básicos, mientras que la programación orientada a objetos resulta más eficiente para proyectos de mayor tamaño o complejidad, donde es necesario gestionar múltiples entidades y facilitar futuras ampliaciones del sistema. Esta experiencia permitió comprender que ambos paradigmas son útiles, pero su elección depende de las necesidades específicas del problema que se desea resolver.
